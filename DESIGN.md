# Street Network Design System (Google Stitch)

This document serves as the machine-readable "source of truth" for the Street Network Admin design system. It ensures that UI components generated by AI maintain the "Cyber/Terminal" aesthetic.

## Design Principles & Tone

- **Identity**: Cyberpunk, Terminal-focused, High-tech, Minimalist.
- **Tone**: Professional yet edgy. It should feel like an advanced operating system or a hacker's terminal.
- **Visual Style**: Dark mode by default, high contrast with vibrant accents, subtle glows, and monospaced technical elements.

## Color Palette & Tokens

### Core Brand
| Variable | Value | Role |
| :--- | :--- | :--- |
| `primary / accent` | `#ff003c` | Primary actions, highlights, branding. |
| `background` | `#0a0a0a` | Global app background. |
| `surface` | `#141414` | Card backgrounds, panels, sidebars. |
| `border` | `rgba(255, 255, 255, 0.05)` | Subtle separators and outlines. |

### Semantic Roles
| Role | Color | Role |
| :--- | :--- | :--- |
| `success` | `#10b981` | Positive states, verified, active. |
| `warning` | `#f59e0b` | Pending states, alerts, cautionary notes. |
| `error` | `#ef4444` | Negative states, closed, critical errors. |
| `muted` | `#64748b` | Less important text, labels. |

## Typography

- **Technical / Code**: `'JetBrains Mono'`, `'Fira Code'`, `monospace`. Used for inputs, database IDs, logs, and technical headers.
- **UI / Content**: Default Sans-serif (Inter preferred). Used for body text and navigation labels.
- **Letter Spacing**: Use `0.02em` for monospaced inputs to improve readability.

## Spacing & Layout

- **Sidebar**: Fixed width (`16rem` / `w-64`), dark background (`#141414`), right border separator.
- **Content Area**: Center-aligned or full-width with responsive padding (`px-6`, `py-8`).
- **Cards**: Rounded corners (`0.5rem` / `rounded-lg`), subtle border.
- **Grid**: Use a 4px/8px baseline grid. Standard gap: `gap-4` or `gap-6`.

## Component Patterns

### 1. Cards (`settings-card`)
- **Background**: `#141414`
- **Border**: `1px solid rgba(255, 255, 255, 0.05)`
- **Shadow**: Subtle inner shadow or no shadow.

### 2. Inputs (`settings-input`)
- **Font**: Monospaced.
- **Background**: `rgba(0, 0, 0, 0.2)`
- **Border**: `rgba(255, 255, 255, 0.05)`
- **States**:
    - **Hover**: Background `rgba(0, 0, 0, 0.3)`, border `rgba(255, 255, 255, 0.1)`.
    - **Focus**: Border `#ff003c`/50, ring `#ff003c`/10.

### 3. Badges (`StatusBadge`, `RoleBadge`)
- **Shape**: Pill-shaped or small rectangles.
- **Style**: Transparent background with low-opacity theme color (`bg-accent/10`), matching border (`border-accent/20`).
- **Effect**: Subtle glow/shadow using matching color (`shadow-[0_0_10px_rgba(accent,0.1)]`).

### 4. Buttons
- **Primary**: Solid `#ff003c`, white text, slight scale effect on click.
- **Secondary**: Transparent with border, subtle hover background.

## UI Constraints (Don'ts)

- **Don't** use light mode colors or white backgrounds.
- **Don't** use soft/rounded default shadows (use color-matched glows instead).
- **Don't** use generic blue or green for primary buttons; use Cyber Red.
- **Don't** use serif fonts.
- **Don't** use large border-radii; keep it at `8px` (medium) or `4px` (small).

## Animations
- **Fade In**: `opacity 0 -> 1`, `translateY(-4px)` (0.2s).
- **Slide Up**: `opacity 0 -> 1`, `translateY(16px)` (0.25s).
- **Pulse**: Used for active status indicators or admin role badges.
